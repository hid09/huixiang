# 回响后台管理系统 PRD

## 1. 项目概述

### 1.1 项目背景
回响项目已上线运行，需要一个独立的后台管理系统，方便管理员查看用户数据、录音记录和生成的日记，评估AI日记生成效果。

### 1.2 技术栈
- **前端**：React 18 + Ant Design Pro 6.x
- **后端**：FastAPI + SQLAlchemy 2.x
- **数据库**：MySQL（与回响共享，只读账号）
- **部署**：同服务器 + Supervisor + Nginx

### 1.3 核心功能

| 模块 | 功能 | 权限 |
|-----|------|------|
| **数据看板** | 核心指标概览、AI 健康度、7 天趋势图 | 超管/普通 |
| **登录** | 账号密码登录 + 退出登录 | 无 |
| **用户管理** | 查看用户列表 + 查看用户详情（录音/日记记录） | 超管/普通 |
| **系统设置** | 管理后台账号 | 超管 |

## 2. 数据库设计

### 2.1 只读数据表（来自回响项目）

#### users 表
```sql
SELECT id, username, phone, created_at FROM users;
```

#### records 表
```sql
SELECT
    id, user_id, content, created_at,
    emotion_type, emotion_score, asr_emotion, mood_tag, input_type, local_date
FROM records;
```

#### diaries 表
```sql
SELECT
    id, user_id, diary_date, emotion_type, mood_tag,
    what_happened, thoughts, small_discovery, keywords, cognitive_change
FROM diaries;
```

### 2.2 自有数据表

#### admin_users 表
```sql
CREATE TABLE admin_users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role ENUM('super', 'viewer') DEFAULT 'viewer',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 3. API 设计

### 3.1 数据看板

| 接口 | 方法 | 说明 |
|-----|------|------|
| /api/admin/dashboard/stats | GET | 获取看板统计数据 |

**返回数据结构：**
```json
{
  "overview": {
    "total_users": 1234,
    "dau_today": 56,
    "records_today": 128,
    "diaries_today": 45
  },
  "ai_health": {
    "success_count": 48,
    "fail_count": 2,
    "fail_rate": 4.0
  },
  "trend_7d": {
    "dates": ["03-03", "03-04", "03-05", "03-06", "03-07", "03-08", "03-09"],
    "new_users": [12, 8, 15, 10, 6, 14, 9],
    "active_users": [45, 52, 48, 60, 55, 58, 56],
    "records": [89, 102, 95, 120, 108, 115, 128],
    "diaries": [34, 40, 38, 45, 42, 44, 45]
  }
}
```

**指标定义：**
| 指标 | 定义 | 数据来源 |
|-----|------|---------|
| DAU | 今日有录音记录的不重复用户数 | `records` 表 `WHERE local_date = 今天` |
| AI 失败率 | 失败数 / 总生成数 × 100% | `diaries` 表 |
| 失败率警告阈值 | > 2% 显示红色 | 前端判断 |

## 4. 页面设计

### 4.1 数据看板页（首页）
- **顶部概览卡片**：总用户数、今日日活、今日录音数、今日日记数
- **AI 健康度卡片**：成功数、失败数、失败率（> 2% 红色警告）
- **7 天趋势图**：折线图展示新增用户、活跃用户、录音数、日记数
- **刷新按钮**：手动刷新数据
- **退出登录按钮**：顶部栏右侧

### 4.2 登录页
- 用户名/密码表单
- 记住我选项
- 登录按钮

### 4.3 主布局
- 左侧：侧边栏菜单（3项：数据看板、用户管理、系统设置）
- 顶部：标题栏 + 刷新按钮（部分页面）+ 退出按钮 + 用户信息
- 中间：内容区
- **退出按钮**：无边框无背景，hover 变红色，位于用户信息左侧

### 4.4 用户列表页
- 搜索框（用户名/手机号）
- 时间范围筛选
- 用户表格（字段：用户ID、用户名、手机号、注册时间、录音数、日记数、操作）
- 导出按钮

- 分页组件

### 4.5 用户详情页（核心变更）
- **基本信息卡片**：用户ID、用户名、手机号、注册时间、累计录音、累计日记、最近活跃、使用天数
- **Tab 切换**：录音记录 / 日记记录
- **录音记录 Tab**：
  - 表格展示：录音时间、内容摘要、ASR情绪、文本情绪、心情标签、操作
  - **情绪对比展示**：ASR 情绪 vs 文本情绪，  - 不一致时高亮标记
  - 分页组件
- **日记记录 Tab**：
  - 卡片列表展示：日期、情绪类型、心情标签、预览内容、录音数、关键词
  - 点击跳转日记详情
  - 分页组件
- **返回按钮**：整合到顶部栏

### 4.6 日记详情页
- **日记基本信息卡片**：日期、用户、情绪类型、心情标签、关键词、录音数
- **录音记录列表**：
  - 时间、内容
  - **ASR情绪 vs 文本情绪对比**（情绪标签）
  - 不一致时红色边框标记
- **日记内容展示**：
  - 今天发生了什么
  - 想法与感悟
  - 小发现
- **返回按钮**：返回用户详情页
- **导航高亮**：侧边栏高亮"用户管理"
- **从用户详情页跳转**：点击日记卡片跳转

## 6. 部署说明

### 6.1 目录结构
```
/opt/echo-admin/
├── backend/
│   ├── app/
│   ├── requirements.txt
│   └── main.py
├── frontend/
│   ├── src/
│   ├── package.json
│   └── nginx.conf
└── supervisor.conf
```
### 6.2 Nginx 配置
```nginx
server {
    listen 80;
    server_name admin.echo-app.com;

    location /api {
        proxy_pass http://127.0.0.1:8100;
    }

    location / {
        root /opt/echo-admin/frontend/dist;
        try_files $uri /index.html;
    }
}
```
