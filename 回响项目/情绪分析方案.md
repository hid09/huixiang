# 回响项目 - 情绪分析方案

## 概述

回响采用**双模态情感识别**：文本情感分析 + ASR 语音情感识别（数据积累中）

当前优先使用文本分析结果，ASR 情感仅存储，后续评估是否需要融合。

---

## 1. 文本情感分析（qwen-plus）

### 1.1 输入类型判断

先判断用户输入属于哪种类型：

| 类型 | 特征 | 处理方式 |
|-----|------|---------|
| A. 情绪表达 | 用户在表达自己的情绪感受 | 正常情绪分析 |
| B. 观点/方法论分享 | 分享经验、想法、知识 | neutral + 平静 |
| C. 事实陈述 | 只说发生了什么，无评价 | neutral + 平静 |
| D. 计划/目标 | 说接下来要做的事 | neutral + 平静 |
| E. 回忆过去 | 回忆以前的经历 | 分析回忆中的情绪 |

**观点分享特征词**：
- "重要的"、"关键是"、"应该"、"建议"、"可以考虑"、"回归本质"、"不是非得"、"还是要"

### 1.2 可选情绪列表

| 大类 | 具体情绪 |
|-----|---------|
| **positive** | 开心、兴奋、满足、期待、感激 |
| **neutral** | 平静、困惑、麻木 |
| **negative** | 疲惫、焦虑、低落、愤怒、失望、恐惧、孤独、压力、烦躁 |

### 1.3 输出结构
```json
{
    "mixed_emotions": {"情绪名": 强度1-10},
    "primary_emotion": "主导情绪",
    "emotion": "positive/neutral/negative",
    "emotion_score": 0.0-1.0,
    "input_type": "情绪表达/观点分享/事实陈述/计划目标/回忆过去",
    "triggers": ["触发事件"],
    "unspoken_need": "潜在需求",
    "energy_level": 1-10,
    "brief_summary": "一句话概括",
    "keywords": ["关键词"],
    "topics": ["话题标签"]
}
```

### 1.4 Bad Case 处理规则

**强调词 ≠ 情绪**：
- "最重要的"、"必须"、"关键" → 强调观点，不是焦虑/压力

**观点分享没有情绪词** → neutral + 平静

**不过度解读**：
- ✅ "还行吧" → 可能有失落
- ❌ "重要的是..." → 不要解读为焦虑

### 1.5 示例

| 输入 | input_type | primary_emotion | emotion |
|-----|-----------|-----------------|---------|
| "产品开发还是要遵循定好的规则" | 观点分享 | 平静 | neutral |
| "Web coding最重要的还是要先做计划" | 观点分享 | 平静 | neutral |
| "今天好累，开了三个会人都麻了" | 情绪表达 | 疲惫 | negative |
| "终于把项目做完了，虽然累但挺有成就感" | 情绪表达 | 满足 | positive（混合：疲惫7+满足8）|

## 2. ASR 语音情感识别（qwen3-asr-flash）

### 2.1 支持的情感类型
| 情感 | 说明 |
|-----|------|
| happy | 愉快 |
| sad | 悲伤 |
| angry | 愤怒 |
| fearful | 恐惧 |
| surprised | 惊讶 |
| disgusted | 厌恶 |
| neutral | 平静 |

### 2.2 数据来源
从 ASR API 返回的 `annotations` 字段中提取：

```json
{
    "choices": [{
        "message": {
            "content": "转写文本",
            "annotations": [{
                "type": "audio_info",
                "emotion": "happy",  // ← 提取这个
                "language": "zh"
            }]
        }
    }]
}
```

### 2.3 存储位置
- 数据库字段：`voice_records.asr_emotion`
- 默认值：`neutral`

---

## 3. 融合策略（待评估）

### 3.1 当前方案
- ASR 情感存储但不用于分析
- 文本情感分析作为主要结果
- 积累数据后评估是否需要融合

### 3.2 可能的融合场景
| ASR 情感 | 文本情感 | 场景描述 | 处理方式 |
|---------|---------|---------|---------|
| happy | 开心 | 一致 | 信心增强 |
| sad | 开心 | 矛盾（强颜欢笑） | 待评估 |
| angry | 平静 | 矛盾（压抑愤怒） | 待评估 |
| neutral | 任意 | 模糊 | 以文本为准 |

### 3.3 后续评估任务
- **数据积累期**：收集 2 周数据
- **分布分析**：统计 ASR 情感和文本情感的分布
- **矛盾场景统计**：统计出现矛盾的频率
- **效果评估**：对比只用文本分析 vs 融合分析的效果
- **决策**：是否需要实现融合逻辑

## 4. 数据库变更

### 4.1 新增字段
```sql
ALTER TABLE voice_records ADD COLUMN asr_emotion VARCHAR(20) DEFAULT 'neutral';
ALTER TABLE voice_records ADD COLUMN input_type VARCHAR(20) DEFAULT '情绪表达';
```

### 4.2 字段说明
| 字段 | 类型 | 说明 |
|-----|------|------|
| asr_emotion | VARCHAR(20) | ASR 语音情感 (happy/sad/angry/neutral等) |
| input_type | VARCHAR(20) | 输入类型 (情绪表达/观点分享/事实陈述等) |
