# 回响项目需求确认清单

> 最后更新：2026-02-28

---

## 一、项目目标

**核心定位**：语音输入 + AI分析的个人成长记录工具，让用户看见自己的变化

**MVP验证目标**：全面验证整体体验，自己先当最真实用户

---

## 二、已确认的需求决策

### 2.1 产品形态
- [x] **MVP形态**：Web先行（响应式Web，1-2周能跑起来，后续可套App壳）
- [x] **输入方式**：语音为主，同时支持文字输入

### 2.2 日记生成
- [x] **生成时间**：固定22:30（简单稳定）
- [x] **当天无记录**：主动提醒 + 生成简单总结（保持仪式感）

### 2.3 复盘周期
- [x] **周回顾**：保留（简洁版）
- [x] **月度复盘**：保留（深度版）
- [x] **季度总结**：保留（趋势分析）
- [x] **年度回顾**：保留（仪式感）

### 2.4 认知变化检测
- [x] **触发时机**：在总结复盘时出现（不需要实时检测）
- [x] **检测间隔**：间隔至少1个月

### 2.5 提醒机制
- [x] **提醒方式**：不做推送提醒，通过首页空状态引导
- [x] **设置页**：精简，只保留"关于回响"

### 2.6 数据存储
- [x] **存储方式**：本地 + 云端（离线可用，数据安全备份）

### 2.7 技术方案
- [x] **AI模型**：通义千问（主力）+ Kimi（深度分析）
- [x] **语音转写**：建议先用通义千问音频能力，效果不好再换专业ASR
- [x] **用量预估**：波动较大
- [x] **成本上限**：200-300元/月

---

## 三、已确认的额外决策

### 3.1 开发节奏
- [x] **MVP形态**：Web先行（响应式Web，1-2周能跑起来，后续可套App壳）
- [x] **开发方式**：用户主导，Claude辅助开发

### 3.2 功能优先级（MVP阶段）
P0功能列表（按PRD执行）：
- [x] 碎片语音输入（支持文字输入备选）
- [x] 每日日记生成（22:30固定时间）
- [x] 四维度分析
- [x] 情绪适配语气
- [x] 首页空状态引导
- [x] 本地+云端存储

P1功能（MVP可选，后续迭代）：
- [x] 认知变化检测（复盘时出现）
- [x] 周/月/季/年度复盘（全部保留）

### 3.3 数据隐私
- [x] **端到端加密**：暂不需要
- [x] **数据导出**：暂不需要

### 3.4 技术方案细化
- [x] **语音转写**：通义千问音频能力优先，效果不好再换专业ASR
- [x] **AI模型分工**：
  - 情感识别：Qwen-Turbo（轻量便宜）
  - 每日日记：Qwen-Plus（平衡）
  - 周回顾：Qwen-Plus
  - 月度/季度/年度复盘：Kimi或Qwen-Max（深度）
  - 认知变化检测：复盘时触发

## 四、技术选型建议

| 类型 | 选择 | 理由 |
|-----|------|------|
| 前端框架 | React + Vite | 生态成熟，开发快 |
| UI组件库 | Ant Design Mobile | 移动端适配好 |
| 后端框架 | Python + FastAPI | AI集成方便 |
| 数据库 | SQLite | 单人版够用，零运维 |
| 前端部署 | Vercel | 免费，自动CI/CD |
| 后端部署 | Render | 免费起步 |
