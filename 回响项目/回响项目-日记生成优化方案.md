# 回响项目 - 日记生成优化方案

## 一、问题回顾

| 问题 | 表现 | 影响 |
|------|------|------|
| 有效记录少 | 3月2日6条录音只有1条有效 | 日记内容空洞 |
| what_happened 和 thoughts 重复 | 同一条记录换说法塞进两个字段 | 用户体验差，显得AI很蠢 |
| 噪音过滤不彻底 | "今天做了"、"小花米" 通过过滤 | 无效信息进入日记 |
| 语句生硬 | "昨天练习强度较大，今天仍感到身体疲惫" | AI腔，不自然 |
| 无兜底策略 | 有效记录很少时硬生成 | 质量不可控 |

## 二、核心优化策略

### 2.1 分层生成策略（核心）

根据有效记录数量，采用不同的生成策略：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         有效记录数 (N)                               │
├─────────────┬─────────────────┬─────────────────────────────────────┤
│   N = 0     │    N = 1-2      │           N ≥ 3                     │
├─────────────┼─────────────────┼─────────────────────────────────────┤
│ 不生成日记   │   简化版日记     │         完整版日记                  │
│ 首页展示关怀  │  只写发生了什么  │     全字段完整生成                   │
└─────────────┴─────────────────┴─────────────────────────────────────┘
```

**详细规则：**

| 有效记录数 | 生成策略 | 处理方式 |
|-----------|---------|---------|
| 0 | **不生成日记** | 首页展示渐进式关怀文案 |
| 1-2 | 简化版 | what_happened: 整合记录<br>thoughts: []<br>small_discovery: null<br>closing: 温柔短句 |
| ≥3 | 完整版 | 所有字段完整生成 |

### 2.2 what_happened vs thoughts 区分规则

**区分原则：**

| 字段 | 含义 | 来源 | 示例 |
|------|------|------|------|
| what_happened | 客观事件 | 用户说的**事情** | "和老朋友吃饭"、"调试录音功能" |
| thoughts | 主观想法 | 用户的**感悟/思考/灵感** | "发现生活不只有工作" |

**处理逻辑：**

```
if 记录是纯事件描述（如"今天吃了火锅"）:
    → 放入 what_happened

if 记录是纯思考感悟（如"突然觉得健康很重要"）:
    → 放入 thoughts

if 记录是事件+思考混合:
    → 事件部分放 what_happened
    → 思考部分放 thoughts
    → 如果不好拆分，优先放 what_happened，thoughts 留空
```

**关键改动：thoughts 可以为空数组 []**
- 不强求必须有 thoughts
- 没有思考就是没有，不要硬编

### 2.3 噪音过滤增强

**现有过滤规则：**
- 文本长度 < 4 字符
- 纯语气词（嗯嗯啊啊）
- 明显测试词（hello、能听到吗）

**新增过滤规则：**

```python
# 1. 语义碎片检测
- 动词后无宾语："今天做了"、"刚刚看到" → 过滤
- 单独的人名/地名/物名："小花米"、"老王" → 过滤
- 无完整句式："因为所以" → 过滤

# 2. 信息量评分
- 对记录进行信息量评分 0-10
- 评分 < 3 的过滤掉

# 3. AI 辅助判断（可选，成本较高）
- 调用轻量模型判断是否有效记录
```

**推荐的实现方式：**
先用规则过滤明显无效的，剩下的交给 AI prompt 里做二次过滤。

### 2.4 Prompt 优化

**优化点：**

1. **增加低信息量场景示例**
```json
// 用户只有1条有效记录
用户记录：
- 15:00 "今天有点累，昨天练得太多了"

输出：
{
    "mood_tag": "需要休息的一天",
    "emotion_type": "negative",
    "keywords": ["休息", "疲惫"],
    "what_happened": ["昨天运动量大，今天感觉累"],
    "thoughts": [],  // 没有，就留空！
    "small_discovery": null,
    "closing": "累了就歇歇，身体会记着你的付出 🌙",
    "tomorrow_hint": null
}
```

2. **明确禁止事项**
```
❌ 不要把同一条记录同时塞进 what_happened 和 thoughts
❌ 没有思考时不要硬编 thoughts
❌ 不要用 AI 腔：综上所述、值得注意的是、建议你
❌ 不要过度解读，用户说什么就是什么
```

3. **语言风格要求**
```
✅ 口语化，像朋友聊天
✅ 简洁有力，一句话说清楚
✅ 温暖但不煽情
```

### 2.5 首页渐进式关怀（新增）

#### 2.5.1 设计理念
- **传递持续陪伴**：当天没有录音时，首页展示关怀文案，告诉用户"我一直在这里"
- **不给压力**：文案极度温和，不是催促用户记录，而是接纳
- **渐进式关怀**：连续多天未记录时，文案会有变化

#### 2.5.2 展示位置
- 首页"今天的记录"区域
- 当天没有录音数据时，展示渐进式关怀文案
- 保持现有样式，保留"—— 回响"

#### 2.5.3 展示效果

```
┌─────────────────────────────────────┐
│  今天的记录                          │
├─────────────────────────────────────┤
│                                     │
│  好几天没听见你的声音了，            │
│  一切都好吗？                        │
│                                     │
│                    —— 回响           │
│                                     │
└─────────────────────────────────────┘
```

#### 2.5.4 渐进式关怀文案

| 连续未记录天数 | 展示文案 | 语气 |
|--------------|---------|------|
| 1天（当天） | 随机选择：<br>• "想说话的时候，我一直在 🌙"<br>• "今天你安静了一天，也许是想休息一下。"<br>• "没有记录的日子，也是生活的一部分。" | 轻松、接纳 |
| 2-3天 | "好几天没听见你的声音了，一切都好吗？" | 关心、不催 |
| 4-7天 | "一周没见，有点想念。不过安静也是好的，照顾好自己。" | 温暖、理解 |
| 8-14天 | "好久不见，希望你一切都好。不是催你，只是想打个招呼。" | 温柔、不打扰 |
| 15+天 | "还在这里，随时等你。" | 简洁、守候 |

#### 2.5.5 连续未记录天数计算
前端调用接口获取连续未记录天数，后端动态计算（不需要新增字段）：
- 查询最近30天的日记记录
- 从今天往前数，遇到第一个有日记的日期停止

def get_consecutive_empty_days(db: Session, user_id: str) -> int:
    """获取连续未记录天数（供前端调用）"""
    from datetime import datetime, timedelta

    count = 0
    today = datetime.now().date()

    for i in range(30):  # 最多回溯30天
        check_date = (today - timedelta(days=i)).strftime("%Y-%m-%d")

        # 检查这天是否有日记
        diary = db.query(DailyDiary).filter(
            DailyDiary.user_id == user_id,
            DailyDiary.diary_date == check_date
        ).first()

        if diary is None:
            count += 1
        else:
            break  # 有日记，停止计数

    return count

### 3.3 噪音过滤函数增强

```python
def _is_valid_record(text: str) -> bool:
    """增强版噪音过滤"""
    # 原有规则 ...

    # 新增：语义碎片检测
    if _is_semantic_fragment(text):
        return False

    # 新增：信息量评分
    if _calc_info_score(text) < 3:
        return False

    return True

def _is_semantic_fragment(text: str) -> bool:
    """检测是否是语义碎片"""
    # 动词后无宾语的模式
    incomplete_patterns = [
        r'^(今天|刚才|刚刚|刚刚)(做了|看了|听了|吃了|去了)$',
        r'^.{1,3}(了|着|过)$',  # 单独的动词后缀
    ]
    for pattern in incomplete_patterns:
        if re.match(pattern, text):
            return True
    return False

def _calc_info_score(text: str) -> int:
    """计算信息量评分 0-10"""
    score = 0

    # 长度加分
    if len(text) >= 10: score += 2
    if len(text) >= 20: score += 1

    # 有完整句子加分
    if '，' in text or '。' in text: score += 2

    # 有情感词加分
    emotion_words = ['开心', '累', '烦', '期待', '失望', '满足', '焦虑']
    if any(w in text for w in emotion_words): score += 2

    # 有事件要素加分
    event_words = ['朋友', '工作', '会议', '吃饭', '项目', '家里']
    if any(w in text for w in event_words): score += 2

    # 纯碎片减分
    if len(set(text)) <= 3: score -= 3

    return max(0, min(10, score))
```
