# 「回响」技术方案设计

> **Slogan**：回响，陪你听见自己

---

## 一、方案概述

基于国产大模型的语音成长日记 App，独立 App 形态，单人版使用。

**核心目标**：让用户通过语音碎片记录，看见自己的变化，辅助自我成长。

---

## 三、技术栈选型

### 3.1 移动端
| 类型 | 选择 | 说明 |
|-----|------|------|
| 跨平台框架 | Flutter | 一套代码双端，性能好 |
| 状态管理 | Riverpod | Flutter 推荐，类型安全 |
| 本地存储 | SQLite | 本地缓存语音和日记 |
| 网络请求 | Dio | HTTP 客户端 |

### 3.2 后端服务
| 类型 | 选择 | 说明 |
|-----|------|------|
| 开发语言 | Node.js / Python | 快速开发，AI 集成方便 |
| 框架 | Express / FastAPI | 轻量级，易扩展 |
| 数据库 | PostgreSQL | 关系型，适合结构化数据 |
| ORM | Prisma / SQLAlchemy | 类型安全，开发效率高 |
| 定时任务 | node-cron / APScheduler | 每日 22:30 生成日记 |

### 3.3 AI 服务（模型分工）

| 任务 | 推荐模型 | 版本 | 理由 |
|-----|---------|------|------|
| **情感识别** | 通义千问 | Qwen-Turbo | 轻量、低成本、高频调用 |
| **每日日记** | 通义千问 | Qwen-Plus | 平衡效果与成本 |
| **周回顾** | 通义千问 | Qwen-Plus | 需要一定理解能力 |
| **月度/年度复盘** | Kimi / Qwen-Max | Kimi / Max | 需要深度分析和长文本 |
| **认知变化检测** | Kimi | Kimi | 理解上下文能力强 |

**模型选择理由**：
- **通义千问为主力**：性价比最高，阿里云服务稳定
- **Kimi 为辅助**：处理需要深度理解的复杂任务
- **轻量模型降本**：情感识别用 Turbo 版，节省成本

### 3.4 其他服务
| 服务 | 提供商 | 用途 |
|-----|--------|------|
| 语音转写 | 阿里云/腾讯云 | 语音转文字 |
| 云存储 | 阿里云OSS/腾讯云COS | 存储语音文件 |
| 推送服务 | 极光推送/APNs | 每日日记推送通知 |

### 4.3 认知变化检测（「过去的我说」）

```
AI 处理流程（Kimi）：
1. 提取当前记录的核心观点
2. 搜索历史记录中的相似话题
3. 对比观点差异
4. 如果差异显著 → 生成「过去的我说」卡片
```

## 五、数据模型设计

### 5.1 核心数据表

```sql
-- 用户表 (单人版只需一条)
CREATE TABLE users (
  id UUID PRIMARY KEY,
  name VARCHAR(50),
  avatar VARCHAR(255),
  timezone VARCHAR(50) DEFAULT 'Asia/Shanghai',
  created_at TIMESTAMP DEFAULT NOW()
);

-- 语音记录表
CREATE TABLE voice_records (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  audio_url VARCHAR(500),          -- 语音文件 URL
  audio_duration INT,              -- 音频时长（秒）
  transcribed_text TEXT,           -- 转写文本
  primary_emotion VARCHAR(20),     -- 主导情绪
  emotion_intensity INT,           -- 情绪强度 0-10
  topic_tags JSONB,                -- 话题标签
  recorded_at TIMESTAMP,           -- 记录时间
  created_at TIMESTAMP DEFAULT NOW()
);

-- 每日日记表
CREATE TABLE daily_diaries (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  diary_date DATE UNIQUE,
  emotion_summary TEXT,            -- 情绪总结
  events_summary TEXT,             -- 事件总结
  thoughts_summary TEXT,           -- 思考总结
  small_discovery TEXT,            -- 小发现
  ai_tone VARCHAR(20),             -- AI 使用的语气
  emotion_journey TEXT,            -- 情绪旅程（多记录时）
  closing_message TEXT,            -- 结束语
  generated_at TIMESTAMP DEFAULT NOW()
);

-- 认知变化记录表
CREATE TABLE cognitive_changes (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  topic VARCHAR(100),              -- 话题
  past_view TEXT,                  -- 过去观点
  past_date DATE,                  -- 过去日期
  current_view TEXT,               -- 现在观点
  current_date DATE,               -- 现在日期
  insight TEXT,                    -- 洞察分析
  created_at TIMESTAMP DEFAULT NOW()
);

-- 复盘报告表
CREATE TABLE review_reports (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  report_type VARCHAR(20),         -- weekly/monthly/quarterly/yearly
  report_date DATE,
  content JSONB,                   -- 报告内容
  created_at TIMESTAMP DEFAULT NOW()
);

-- 统计表（预计算，提升查询性能）
CREATE TABLE user_stats (
  user_id UUID PRIMARY KEY REFERENCES users(id),
  total_record_days INT DEFAULT 0,        -- 总记录天数
  total_voice_count INT DEFAULT 0,        -- 总语音条数
  continuous_days INT DEFAULT 0,          -- 连续记录天数
  longest_continuous_days INT DEFAULT 0,  -- 最长连续天数
  last_record_date DATE,                  -- 最后记录日期
  updated_at TIMESTAMP DEFAULT NOW()
);
```

### 5.2 统计逻辑说明

#### 记录天数
```sql
-- 统计有记录的日期数量
SELECT COUNT(DISTINCT recorded_at::date)
FROM voice_records
WHERE user_id = {user_id}
```

#### 连续记录天数
从今天往前推，连续每天都记录的最长天数。中断后重新计算。

#### 本月记录占比
```javascript
const percentage = (本月记录天数 / 今天日期) * 100
// 示例：23/28 = 82%
```

## 八、成本估算（单人版）

| 项目 | 月成本 | 说明 |
|-----|-------|------|
| 通义千问 API | ~50-150元 | Qwen-Turbo + Qwen-Plus |
| Kimi API | ~50-100元 | 用于深度分析和认知变化 |
| 语音转写 | ~30-100元 | 阿里云/腾讯云 |
| 云服务器 | ~50-100元 | ECS/CVM |
| 云存储 | ~10-50元 | OSS/COS |
| 推送服务 | ~0-50元 | 极光免费额度基本够用 |
| **合计** | **约 100-450元/月** | 根据使用量浮动 |

**成本优化建议**：
- 优先使用通义千问，性价比最高
- 情感识别使用 Qwen-Turbo，成本极低
- 只在必要时调用 Kimi

## 九、技术风险与应对

| 风险 | 应对方案 |
|-----|---------|
| AI 模型不稳定 | 接入多个模型，支持降级切换 |
| 语音转写失败 | 保留原始音频，支持重试 |
| 推送送达率低 | 支持多种推送方式 |
| 数据安全 | 数据加密存储，定期备份 |
